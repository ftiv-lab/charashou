# ハンドオフ 0013b — 右インスペクタを写真/校章/印へ拡張 ＋ 発見性の改善

- 作成：Claude / 2026-06-13
- 状態：完了

## 目的
0013で入れた右インスペクタを、**テキスト以外（写真・校章・印）でも開く**ようにし、ユーザーが**「クリックで編集できる」と気づける**ようにする（発見性）。

## 背景
- 0013ではテキスト選択時のみインスペクタが開く。写真/校章/印は選択（Transformer表示）はできるが**インスペクタが出ない**。
- 実機で「出し方が分からない」が発生＝**未選択時は手掛かりゼロ＋ホバーが`move`カーソルのみ＋下のヒント文に案内なし**＝発見性の問題（※リロードで直ったのは古いページのキャッシュ。コード不具合ではない）。

## やること

### A. インスペクタを全編集要素に対応（text / image / crest / seal）
- `handleSelectionChange`：**編集可能要素（`isEditableElement`）の選択でインスペクタを開く**（テキスト限定をやめる）。
- RightInspector を**要素種別で出し分け**（Polotno式）。
  - **共通ブロック「位置・サイズ」**：X / Y / 幅 / 高さ。
  - **種別ごとの追加**：
    - **text**（既存）：内容 / 文字サイズ / 文字色。
    - **写真(image)**：ズーム / 横位置 / 縦位置（＝`photo` state を `onPhotoAdjustment` で）。
    - **校章(crest) / 印(seal)**：位置・サイズのみ（色＝全体テーマ、内容生成は0015）。
- 見出し「選択中：○○（種類）」：text=フィールド名、image=写真、crest=校章、seal=印。
- **単一原本厳守**：写真調整＝`onPhotoAdjustment`、要素の位置/サイズ＝`onElementChange`、テキスト＝既存ハンドラ。**新規 state を作らない**。

### B. 発見性の改善
- 下のヒント文を更新：「**カード上の要素（文字・写真・校章・印）をクリックすると、右で詳細編集できます**」。
- **ホバーで編集可能要素を薄くハイライト**（細い枠線など）＋カーソルは「クリックできそう」な見た目に（`move`一辺倒をやめ、クリック＝編集だと伝わるように）。

## アクセシビリティ（0013継続）
- 開閉トグルに `aria-expanded`/`aria-controls`、選択時に `aria-live` 通知（「写真を選択しました」等）、**クリック選択でフォーカスはキャンバス維持**、Esc＝まず選択解除。

## テスト/検証
- `npm run check` / `check:e2e` 緑。
- 既存のテキストインスペクタE2Eは維持。**E2E追加**：写真要素をクリック→インスペクタが開く→ズーム変更→カード反映（＋校章クリックで位置・サイズが出る、を確認）。
- 完了後 `npm run shots` でスクショ再生成（写真/校章選択時の図など）。

## 受け入れ基準
- カード上の **文字・写真・校章・印** をクリックで選択→右インスペクタが**スライドで開く**。
- 種別ごとに正しい項目が出る（写真＝ズーム/横/縦＋位置サイズ、校章/印＝位置サイズ、テキスト＝内容/サイズ/色＋位置サイズ）。変更が**即カード反映**（state は1つ＝二重管理なし）。
- 下のヒントに要素クリックの案内があり、編集可能要素は**ホバーで反応**して「押せる」と分かる。
- 既存（4タブ・保存/JSON・Undo/Redo・PNG・テキストインスペクタ）が壊れていない。
