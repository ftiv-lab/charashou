# ハンドオフ 0014a — 装飾を“パラメータ生成”化 ＋ プリセット（素材スロットの最小コア）

- 作成：Claude / 2026-06-13
- 状態：完了

## 目的
テンプレの“素材（校章・印・透かし・背景地紋）”を、固定描画から**パラメータ生成（手続き的）**に変える＝**テーマ連動・保存可能・再利用可能・3倍書き出しでも綺麗・著作権クリーン**。複数テンプレ(0014b)の土台。今ある学生証の完成度も同時に上がる。

## 背景（council：Claude+ChatGPT 一致）
- 校章/印/透かし/地紋は**AI画像より手続き的生成が最適**（色追従・再現性・権利クリーン・テンプレ流用）。
- ただし**素材システムを全部先に作らない**。**「素材の“枠と生成”の最小コア」だけ先**→ 0014b 複数テンプレ → 0015 生成の豪華版（アップロード管理/AIプロンプト補助/「素材」タブ独立）。
- 背景は本格ギョーシェ不要。**世界観寄り**（かわいい地紋/魔法陣風/サイバー線/ギルド紋様）でよい。

## アーキテクチャ判断（このスライス）
- **要素モデルを維持**し、`crest`/`seal`/`watermark` 要素＋新規 `pattern`(背景地紋) を**パラメータ駆動**にする。`Template.category` や `assetSlots` 抽象は **0014b で**導入（ここではやらない）。
- 生成パラメータ＝判別ユニオン（例。命名はコードに合わせて可）：
  - `monogramCrest`：initials / shape(circle|shield|diamond) / strokeColor / fillColor
  - `roundSeal`：outerText / centerText / color
  - `repeatTextWatermark`：text / angle / opacity
  - `pattern`：kind(stripe|dot|wave|rosette) / color / opacity
- 色は**未指定なら theme に追従**（crestAccent 等）。

## やること
1. 上記の生成パラメータ型を `template.ts` に追加し、`crest`/`seal`/`watermark`/`pattern` 要素が保持できるようにする。
2. **CardPreview を生成パラメータから手続き描画**（既存の固定描画を置換）。背景 `pattern` も追加（frame系＝`listening=false`）。
3. **デザインタブにプリセット選択**：校章/印/透かし/背景それぞれ**3個ずつ**（クリックで該当要素のパラメータを差し替え）。
4. **Zod schema / 保存 / JSON / Undo-Redo / PNG書き出し** に生成パラメータを反映（往復・履歴・3倍出力が壊れない）。
5. **学生証テンプレの既定**を新パラメータで再現（見た目維持〜微改善）。

## アクセシビリティ / 既存維持
- プリセットはボタン/ラジオで選択でき、キーボード操作可。
- 既存（4タブ・右インスペクタ・保存/JSON・Undo/Redo・PNG・スクショ）が壊れない。

## テスト/検証
- `npm run check` / `check:e2e` 緑。
- **単体**：生成パラメータ→描画用の純関数（あれば）／schema往復／プリセット適用。
- **E2E**：プリセットを変えるとカードが変わる→保存/リロードで復元→PNG書き出しに反映。
- 完了後 `npm run shots` 再生成（校章/印/背景プリセット違いが分かる図）。

## 受け入れ基準
- 校章/印/透かし/背景が**パラメータ生成**で描かれ、**デザインタブでプリセット3種ずつ選べる**＝即カード反映。
- 保存/JSON/Undo-Redo/PNG が生成パラメータ込みで正しく動く。
- 学生証の見た目が維持（または微改善）。

## スコープ外（厳守・後続）
- `Template.category` / `assetSlots` 抽象、複数テンプレ、テンプレ切替（→ 0014b）。
- 本格ギョーシェ、素材アップロード管理、AIプロンプト補助、「素材」タブ独立、5種以上テンプレ、販売パック（→ 0015）。
- 右インスペクタでの生成パラメータ細編集（ここは**プリセット選択のみ**。細かいパラメータは後続）。
- 全体テーマの右複製・レイアウト刷新・右インスペクタのかぶり対策（別ポリッシュ）。
