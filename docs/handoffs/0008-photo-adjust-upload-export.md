# ハンドオフ 0008 — 写真の調整（ズーム/位置）＋ アップロード検証 ＋ 保存中表示

- 作成：Claude / 2026-06-12
- 状態：未着手

## 目的
キャラ絵をきれいに入れられるようにする：**写真のズーム/位置調整（顔が切れない）**＋**安全なアップロード（形式/サイズ検証）**＋**書き出しの保存中表示**。ChatGPTレビューで最優先級だった項目。

## 前提（現状）
- 写真は App state の `photoDataUrl`（string）。`CardPreview` の `image` 要素内で `coverCrop`（中央固定）で表示。写真枠（位置/サイズ）は0007でドラッグ/リサイズ可能。
- 書き出し `export.ts`：`editor-ui` レイヤーを隠して `stage.toDataURL({pixelRatio:3})`。
- パネル `PropertyPanel`：テーマ＋項目（内容/スタイル）＋写真ファイル選択。

## A. 写真のズーム/位置（クロップ調整）
- App の写真状態を **`{ dataUrl, zoom, offsetX, offsetY }`** に拡張（`zoom` 1〜3、`offsetX/offsetY` -1〜1 のパン率、既定 zoom=1/offset=0）。
- パネルに**「写真」セクション**：ファイル選択 ＋ **ズーム slider** ＋ **横位置 / 縦位置 slider**。
- `CardPreview`：固定 `coverCrop` を **zoom/offset から算出するクロップ**に置換（枠＝`width×height` に対し、zoomで拡大・offsetでパン、はみ出しはクランプ）。
- **枠の移動と画像のパンは別操作**：キャンバスのドラッグは**枠の移動**（0007のまま）、画像のズーム/パンは**パネルのslider**。
- 「既定に戻す」はテンプレ（カード）を戻すが、**アップロード画像は消さない**（写真はユーザーcontent）。zoom/offset初期化ボタンは任意。

## B. アップロード検証
- `accept="image/png,image/jpeg,image/webp"`（SVG等は受けない）。
- **サイズ上限（例 10MB）超過＝エラー表示**、画像でない/対象外形式＝エラー表示。
- エラーは**画面に明示**（入力の近く）＋ **`aria-live`** で読み上げ。成功時はクリア。
- 検証は FileReader 実行**前**に行う（巨大ファイルでブラウザが重くなるのを防ぐ）。

## C. 書き出しの保存中表示 ＋ toBlob
- 「PNGで保存」ボタンを処理中 **「保存中…」＋disabled**（連打防止）。完了/失敗を **`aria-live`** で通知。
- `export.ts` を **Blobベース**へ：`stage.toBlob({...})`（or toDataURL→Blob）＋ `URL.createObjectURL` → ダウンロード → `URL.revokeObjectURL`（メモリ効率）。**`editor-ui` レイヤーを隠して書き出す現行処理は維持**。`pixelRatio:3` 維持。

## テスト/検証
- `npm run check` 緑・`npm run check:e2e` 緑。
- **純ロジック（クロップ算出・アップロード検証）を単体テスト**に。保存中表示・エラー表示は最小E2E/コンポーネントテストで。

## 受け入れ基準（完了確認コマンド）
- 写真の **ズーム/横位置/縦位置** を変えると**プレビュー即反映**（枠内でクロップ）。
- **対象外形式/サイズ超過でエラー表示＋読み上げ**、正常画像は表示。
- **保存中はボタンが「保存中…」＋無効**、完了/失敗が通知される。
- PNG（3倍）は従来どおり・**選択枠/ガイドは写らない**。
- 既存（編集/テーマ/自由配置/写真表示・PNG）が壊れていない。
