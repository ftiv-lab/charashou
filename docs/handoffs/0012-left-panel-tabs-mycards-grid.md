# ハンドオフ 0012 — 左パネルのタブ化 ＋ マイカードのサムネグリッド化（レイアウト整理・Step 1+2）

- 作成：Claude / 2026-06-13
- 状態：未着手

## 目的
左の縦長パネルを**タブ構成**に再編し、**保存カードをサムネイル・グリッド**にする。＝「縦長・保存カード一覧が見にくい」を**最短・低リスク**で解消。**新機能は足さない＝既存コントロールの再配置のみ**（右インスペクタ/3ペイン/複数テンプレは後続）。

## 背景（council結論）
Claude＋ChatGPTとも「編集フォーム／インスペクタ／保存カード／将来テンプレ を1パネルに混在」が問題、と一致。安全な段階導入の **Step1(タブ化)＋Step2(カード→ライブラリ・グリッド)** を本スライスで実施。

## 前提（現状）
- `App.tsx`：topbar(タイトル/Undo/Redo/PNG保存) ＋ `PropertyPanel`(左) ＋ stage(中央)。
- `PropertyPanel`：全体テーマ ＋ 項目(各fieldの内容入力＋「スタイル」開閉) ＋ 写真 ＋ `StoragePanel`(保存/保存カード一覧/読込/複製/改名/削除/JSON/自動保存表示)。
- 保存カードは0011でサムネ生成済み（`thumbnail`）。

## やること（再配置のみ）
左パネルを**タブUI**にする：
- **内容**：各fieldの内容入力（＋現状の「スタイル」開閉もここに維持）。＝現「項目」セクション。
- **デザイン**：全体テーマ（cardBg/帯/文字/校章アクセント/ベースフォント/透かし）。＝現「全体テーマ」セクション。
- **写真**：写真アップロード＋ズーム/横位置/縦位置。＝現「写真」セクション。
- **マイカード**：`StoragePanel` を**サムネイル・グリッド**に。各カード＝サムネ＋名前＋更新日時＋［開く］［複製］［改名］［削除］。＋「保存」ボタン・JSON書き出し/読み込み・自動保存インジケータ。

## 制約
- **左に置いたまま**（右インスペクタ化・3ペイン化・`selectedElementId` の App 持ち上げは 0013）。タブは左パネル内で切替。
- **既存ロジックは不変**（編集/テーマ/写真/保存/JSON/Undo-Redo/書き出し/履歴）。**見た目・配置の再編のみ**。
- 既定の選択タブ＝「内容」。タブ状態は local state でよい。

## アクセシビリティ
- タブは ARIA（`role="tablist"/"tab"/"tabpanel"`・`aria-selected`・`aria-controls`・矢印キーでタブ移動）。

## テスト/検証
- `npm run check` / `check:e2e` 緑。
- 既存E2E/単体が**タブ内の入力に触れる場合、対象タブへ切替してから**操作するよう追従（不活性タブの要素は不可視/未描画になり得る）。
- 完了後に **`npm run shots`** でスクショ再生成（パネルがタブUIに変わるため README を最新化）。

## 受け入れ基準（完了確認コマンド）
- 左パネルが **内容 / デザイン / 写真 / マイカード** のタブになっている。
- **マイカード＝サムネイル・グリッド**（保存カード）＋保存＋JSON入出力＋自動保存表示。
- 既存機能が**すべて動く**（内容/スタイル/テーマ/写真編集・保存/読込/複製/改名/削除・JSON・Undo/Redo・PNG書き出し）。
- タブがキーボード/ARIA対応。
- `npm run check` / `check:e2e` 緑、`npm run shots` 実行済み。
