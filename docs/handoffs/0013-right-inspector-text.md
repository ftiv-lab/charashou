# ハンドオフ 0013 — 右インスペクタ（選択追従・テキストのみ・スライド開閉）

- 作成：Claude / 2026-06-13
- 状態：未着手

## 目的
「**キャンバスで要素を選ぶ→右でその要素を編集する**」感覚を作る（Figma/Polotno方式の最小版）。**常時3ペインにはしない**。**選択時だけ開くスライド式の右インスペクタ**から始める。

## 背景（council：Claude+ChatGPT 一致）
- 右インスペクタの方向は正しい。ただしスクショ（1440px / 左約360px / 横長カード）を見ると**常時の右固定列はカードが窮屈**＝「**選択時に開く・閉じられる**」スライドが安全（＝ユーザーの“使わない時はひっこむ”直感どおり。Figmaの最小化挙動と同じ）。
- **一度に全部やらない**：写真/校章/印/複数選択/ピン/永続化/スマホは後回し。**最初はテキスト要素のみ**。
- 左の「項目別スタイル」は**今は残す**（いきなり剥がすと導線が壊れる）。役割整理は後続で段階的に。
- **二重state禁止**：UIが左右2箇所でも state は1つ（template の fields/elements）。右も**既存の dispatch ハンドラを再利用**するだけ。

## やること（最小1スライス）
1. **`selectedElementId` を App 最上位へ持ち上げ**（現在は CardPreview ローカル）。選択は**履歴外**（0010方針を継続）。
2. **キャンバスのテキスト要素クリックで選択**／**空白クリック・Esc で選択解除**。
3. **RightInspector を追加**＝**選択時に開くスライドパネル**（閉じるボタンで畳む）。
4. **テキスト要素のみ対応**。編集項目は **内容 / X / Y / 文字サイズ(fontSize) / 文字色(color)** だけ。
   - すべて**既存の状態を更新**（`onFieldValueChange` / `onFieldStyleChange` / `onElementChange` を再利用）。**新規 state を作らない**。
   - `fieldKey` 付きテキストは field 値/スタイルを、その他は element を更新（単一原本を維持）。
   - パネル上部に「**選択中：氏名（テキスト）**」の見出し。
5. **未選択時**：パネルは閉じる、または「カード全体は左の“デザイン”タブで」程度の案内（**全体テーマの重複コントロールは置かない**）。
6. **アクセシビリティ**：
   - 右パネル開閉トグルに **`aria-expanded` / `aria-controls`**（Disclosure パターン）。
   - **クリック選択でフォーカスはキャンバスに残す**（右へ飛ばさない）。`aria-live="polite"` で「氏名を選択しました」程度の通知。
   - **Esc＝まず選択解除**（パネルを閉じるのは二段目でよい）。

## レイアウト
- 既存の topbar / 左パネル / 中央 stage に、**右へ重ねて出るスライド**で RightInspector を追加。**常時固定の3列にはしない**（中央カードを狭めないため）。
- デスクトップ前提。狭い画面の最適化（ドロワー/ボトムシート）は後回し。

## テスト/検証
- `npm run check` / `check:e2e` 緑。
- 選択・解除・スライド開閉は**純ロジック/状態**を中心に単体。Konva 上の選択は E2E で。
- **E2E 1本**追加：氏名テキストをクリック→右インスペクタが開く→文字サイズを変更→カードに反映→Esc で選択解除（→未選択時の挙動）。
- 完了後 `npm run shots` でスクショ再生成（右インスペクタ表示の図を追加 or 更新）。

## 受け入れ基準
- キャンバスのテキストをクリックで選択、右インスペクタが**スライドで開く**。閉じるボタン/Esc で畳める・解除できる。
- 右で **内容 / X / Y / 文字サイズ / 文字色** を変えると**即カード反映**（state は1つ＝二重管理なし）。
- 開閉トグルに `aria-expanded`/`aria-controls`、選択時に `aria-live` 通知、**クリック選択でフォーカスはキャンバス維持**。
- 既存（左4タブ編集・保存/JSON・Undo/Redo・PNG・スクショ）が壊れていない。
