# キャラ証 — 設計メモ（SPEC）

元の壁打ち仕様：`E:\IdeaLab\2026-06-12-card-maker-spec.md`（要素分解の詳細）。ここはその実装版の要点。

## カードの構造
- `#card`（680×430px）はKonva Stageの表示枠。`Template.elements` の座標要素をLayerへ順番に描画する：
  1. 背景／ピンク帯
  2. 透かし（薄い繰り返し文字・回転）
  3. 校章／写真／各テキスト／印
- 出力はKonva Stageの `toDataURL({ pixelRatio: 3, mimeType: "image/png" })` で2040×1290 PNG化。

## フィールド（差し替え対象）
| key | 内容 | 種別 |
|---|---|---|
| schoolName / schoolRoman | 学校名・ローマ字 | text |
| title | タイトル(学生証) | text |
| grade | 学年 | text |
| name / nameRoman | 氏名・ローマ字 | text |
| birth / expiry | 生年月日・有効期限 | text |
| statement / issuer | 証明文・発行者 | text |
| photo | 顔写真 | image(upload: PNG/JPEG/WebP、10MB以下) |
| crest / seal | 校章・印 | Konva図形（将来 image 化） |

- 内容・テーマ・スタイル・座標の単一ソースは `src/card/template.ts` の `Template`。Canvas内テキストはDOM要素ではない。

## スタック（決定 2026-06-12）
**Vite ＋ React ＋ TypeScript**（クライアント完結・バックエンド無し）。商品化＝Booth配布(zip)／Tauri(.exe)／テンプレ販売。詳細は `AGENTS.md`。

## 安全設計（公開・配布する場合）
ID系メーカーは偽造に使われ得るため、公開時は最初から健全側に倒す（由来＝マルチモデルcouncilでChatGPTが偽造リスクを指摘・採用）：
- **実在の学校名/ロゴは使わない**（架空前提・既定テンプレも架空のコロンド女子学院）。
- 必要に応じ **`SAMPLE`/`架空`/`FICTIONAL` 透かしオプション**（将来）。
- 名前 **「キャラ証」が既に“キャラ＝架空のID証”を示す**＝健全な方向（「学生証メーカー」より誤解されにくい）。
- 写真アップロードのサイズ/MIME検証（将来・公開機能を作る場合）。

## 編集の土台＝react-konva（決定 2026-06-12）
カードを**Canvasの要素群**に作り直し、**自由配置＋スナップ/ガイド**の本格エディタへ（CLIP STUDIO/Canva的）。理由＝text-alignでは「揃える箱」が見えず位置の自由度が無い。書き出しもKonvaで安定。テストはカードがDOM→Canvasになるため、パネルはDOM維持＋スクショ/書き出しE2Eへ寄せる。

## 段階プラン（カスタム化）
- **0005（済）**：テンプレJSON＋プロパティパネル＋全体テーマ。
- **0006（済）**：カード描画を**Konvaへ移行**（パリティ＋Konva書き出し・対話まだ・テスト方式移行）。
- **0007（済）**：**自由配置エディタ**（内容要素の選択/ドラッグ/変形/スナップ/ガイド）。背景/帯/透かしは固定。ルーラーは未実装。
- **0008（済）**：写真のズーム/横位置/縦位置slider＋アップロード検証(形式/10MB/aria-liveエラー)＋保存中表示＋toBlob書き出し。キャンバス上の画像パンは未実装。
- **0009**：印・モノグラム校章ジェネレーター（＋アップロード）。
- **0010**：複数テンプレ（学園/魔法学校/ギルド/サイバー/VTuber/ステータス…）＋切替＋JSON保存/読込＋localStorage。
- 後：ビジュアル回帰テスト／WebP・PDF／Tauri(.exe)。
